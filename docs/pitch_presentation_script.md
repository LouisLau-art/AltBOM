*   **【画面】**：*第二页：三栏红色/黄色警示卡片，揭露供应链暴雷、寻源门槛和烧板风险*
*   **【情绪建议】**：沉稳、指出痛点时要有痛惜和急迫感。

> **🎤 [逐字稿台词]**：
> 
> 在传统的硬件研发流程中，有三堵高墙将研发与采购死死阻隔：
> 
> *   **第一是供应链暴雷**：欧美芯片巨头停产、限售、缺货频发，企业面临极大的合规和断供威胁。
> *   **第二是寻源成本高昂**：市场上国产平替芯片有成千上万种。一个板子上百种元器件，工程师要翻看几百份几千页的 Datasheet 去人工比对，效率极其低下。研发与采购处于严重的信息孤岛。
> *   **第三是盲目换料的巨大风险**：仅凭芯片名字相似或依靠通用大模型的模糊回答，极易忽略引脚重定义或电气特性的细微差异。最终的代价，就是电路板在焊接通电的瞬间，冒烟烧毁！
> 
> 为了解决这一痛点，我们开发了 AltBOM。

### 📐 Slide 5：AI FAE 设计避坑指南 (限时：60秒)
*   **【画面】**：*第五页：左右两栏技术细节卡片，平铺物理改板和固件驱动差异建议*
*   **【情绪建议】**：极其专业、严谨、打消专家评委对于替换安全性的最大疑虑。

> **🎤 [逐字稿台词]**：
> 
> 很多懂行的硬件评委可能会问：你们换了料，板子能直接用吗？
> 
> 这正是 AltBOM 最具竞争力的地方。我们的系统直接在 Dashboard 中平铺了 **AI FAE 设计防雷及固件建议**：
> 
> *   **在硬件方面**：我们明确指出，虽然换稳压片省钱，但封装从 SOT-223 变成了 SOT-23，脚位一致但焊盘变小，必须通知硬件工程师微调 PCB Layout。
> *   **在固件方面**：我们提醒软件组，兆易创新的 Flash 制造商 ID 是 `0xC8`，如果原代码中写死了校验华邦的 `0xEF`，必须增加兼容代码，否则系统上电会死机自检不通过。同时，蓝牙模块替换后，AT 指令的初始化速率也需要从 38400 修改为 9600。
> 
> AI 把这些以往需要工程师看几周手册才能踩完的坑，在一秒钟内平铺直叙地列了出来。
