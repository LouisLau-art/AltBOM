# AltBOM 黑客马拉松路演答辩 Q&A 评估卡 (FAE & Pitch QA Cards)

在黑客松的路演答辩环节，评委的提问通常非常务实且尖锐。以下是根据电子研发与供应链真实痛点整理的 **5 大核心必问问题** 及其 **满分回答预案**。

---

### 卡片 1：防烧板与硬件引脚兼容性（硬件专家必问）

> **💡 评委问题**：
> “大模型常常会有幻觉，你怎么保证 AI 给你推荐的替代芯片是绝对安全的？如果引脚定义变了或者电压不对，一通电把电路板烧了或者芯片冒烟，这个责任谁负？”

#### 🎯 满分答辩预案：

1. **强调“双通道混合检索”架构**：
   * 我们的系统不是纯靠 GPT/大模型去猜芯片。我们内置了 **Hybrid Search 混合模型**。
   * **向量通道** 负责模糊匹配功能描述（语义相似度）；**关键字通道** 强制对物理封装（如 `lqfp48`、`sot-23`）、工作电压（如 `3.3V`）等进行硬性字符串比对。参数有一丁点不对齐，AI 评分就会骤降，绝不强行匹配。
2. **强调“白盒透明化与工程师最终审核”**：
   * 我们绝对没有做成黑盒。Dashboard 的表格中，我们开辟了 `AI FAE 设计防雷及固件建议` 一栏，将引脚兼容性（如 100% 直替还是引脚间距公差）、改板改焊盘风险（如 SOT-223 换 SOT-23 需要微调 PCB Layout）清清楚楚铺平展示给工程师。
   * AI 定位为“**辅助决策的雷达**”，把所有差异白盒化推给工程师签字，将改板风险降到零。

### 卡片 2：商城价格的实时性与时延（供应链/采购评委关注）

> **💡 评委问题**：
> “电子元器件价格是随时波动的，甚至每小时都在变。你们这套系统显示的价格如果是静态的，那还有什么参考价值？如果是联网的，大模型联网搜索太慢了，怎么解决时延问题？”

#### 🎯 满分答辩预案：

1. **大模型联网搜索插件 (Web Search API)**：
   * 我们后端完全摒弃了静态数据库。当提交 BOM 分析时，后端会直接调用火山引擎大模型 API 的 **联网搜索功能 (Web Search Tool)**。它会现场去立创商城、得捷等平台的公开现货数据页抓取**那一分那一秒**的真实报价与库存。
2. **三级降级与缓存机制 (三级保障)**：
   * **一级**：真联网，拉取最真实数据。
   * **二级（8秒超时保护）**：如果黑客松现场网络拥堵，我们代码内置了 `timeout=8`。若超过 8 秒未返回，瞬间无感降级为**高保真本地默认缓存数据库**，确保台上的演示绝不卡死。
   * **三级（一键穿透验证）**：我们在表格里为每一颗芯片都加上了 **`🌐 官方商城链接`**。评委如果不信，可以当场扫码或点击，页面会直接穿透进入立创商城的官方型号搜索页，实时对比，用真凭实据消除疑虑。

### 卡片 3：与现有硬件开发流的接轨（产品落地价值）

> **💡 评委问题**：
> “硬件设计是一个长周期的严谨流程，你们这个好看的毛玻璃网页，怎么融入到我们已有的开发、改板和采购流程中？”

#### 🎯 满分答辩预案：

1. **设计了采购专用的 ECN (工程变更单) 双导出闭环**：
   * 我们不是一个“只能看”的网页，它能直接产出两份可以马上交接工作的**物理文件资产**：
2. **一键导出优化后采购 BOM (CSV)**：
   * 自动过滤并替换被 AI 优化后的 6 颗芯片型号与采购描述，保留其他阻容。采购人员拿到这个 CSV 即可一键上传进商城或公司 ERP 完成批量下单，省去人工改单的时间。
3. **一键生成技术变更报告 (HTML/PDF)**：
   * 包含详细降本图表、AI 避坑防雷指南。工程师直接在浏览器中 `Ctrl+P` 就能保存成精美的 PDF 盖章变更单，发给 EE 硬件组去微调 PCB Layout，发给软件组去修改底层驱动代码，研发和采购瞬间协同。

### 卡片 4：真实替代案例的底层细节（嵌入式/硬件细节提问）

> **💡 评委问题**：
> “你说你把 Winbond 的 Flash 换成了国产兆易创新 (GD) 的，或者把 HC-05 蓝牙换成了 JDY-31，底层软件到底要改什么？别只跟我吹降本。”

#### 🎯 满分答辩预案：

我们对平替物料的底层物理和固件差异做到了极其严谨的研判：

* **SPI Flash 平替的 JEDEC ID 坑**：
  Winbond 的 JEDEC 制造商 ID 是 `0xEF`，而 GD25Q64 的制造商 ID 是 `0xC8`。如果 MCU 原固件驱动中有自检读取 ID 并判断是否等于 `0xEF` 的校验代码，必须修改驱动添加对 `0xC8` 的兼容，否则系统上电会直接死机报错。
* **LDO 电源平替的封装与待机电流**：
  原 AMS1117 是 SOT-223 封装，LDK130 是 SOT-23。虽然脚位顺序一样，但焊盘大小不同，硬件工程师**必须重画 PCB 封装**。但换料后静态电流从 5mA 降到 120µA，对于智能手表这类穿戴设备，待机时长能直接延长数周！
* **蓝牙平替的 AT 指令波特率**：
  经典蓝牙模块 HC-05 的 AT 配置模式默认是 38400 波特率，而 JDY-31 默认波特率是 9600。如果不修改 MCU 串口初始化的发送速率并重写其 AT 指令解析，无线连接就会死锁。

### 卡片 5：商业价值与未来愿景（评委大格局提问）

> **💡 评委问题**：
> “你们这个项目的商业空间有多大？这只是个小工具，怎么能成为一个有壁垒的产品？”

#### 🎯 满分答辩预案：

1. **从“单机雷达”向“云端企业物料智能池”演进**：
   * AltBOM 现在的形态是一个小工具。但在未来，它可以发展为**企业级智能物料管理平台 (IPD-PLM 辅助器)**。
   * 我们可以在企业内部建立“公共物料池”与“多源替代料库”。当新的研发项目启动时，AI 可以实时审核新板的 BOM，强制要求复用库里的低成本、货源稳定的型号，在设计阶段就把“供应链断货风险”和“无效物料种类”消灭掉。
2. **AI 重塑研发与采购的边界**：
   * 传统的硬件开发中，研发不知道采购拿到的价格，采购看不懂研发的 Datasheet，两者是割裂的。
   * AltBOM 利用 AI 联网检索与参数特征匹配，打破了这种信息孤岛，缩短了硬件产品的研发上市周期，提高了中国电子制造和研发链条的敏捷度。
